STORAGE IMAGE FIX - Quick Guide
================================

PROBLEM: All images showing ❌ Failed to load
CAUSE: Missing symlink from public/storage → storage/app/public

FILES TO UPLOAD:
1. fix-storage-link.php (from d:\xammp\htdocs\aprati_web_backup\aprati_backend\public\)
   Upload to: /home/apratifoodscom/public_html/sdev.apratifoods.asia/

2. test-images-complete.html (already created)
   Upload to: /home/apratifoodscom/public_html/sdev.apratifoods.asia/

STEPS:
------
1. Upload fix-storage-link.php via cPanel File Manager
2. Visit: https://sdev.apratifoods.asia/fix-storage-link.php
3. Click "Copy Files Now" button (if symlink fails)
4. Visit: https://sdev.apratifoods.asia/test-images-complete.html
5. All images should show ✅ Loaded

WHY THIS HAPPENS:
-----------------
Laravel stores files in: storage/app/public/brands/logos/xxx.png
But serves them from:    public/storage/brands/logos/xxx.png

This requires a symlink (public/storage → ../storage/app/public)
Shared hosting often disables symlinks, so we copy files instead.

MANUAL FIX (if script doesn't work):
-------------------------------------
Use cPanel File Manager:

1. Go to: /home/apratifoodscom/public_html/sdev.apratifoods.asia/storage/app/public/
2. Select ALL folders (brands, products, etc.)
3. Copy them
4. Go to: /home/apratifoodscom/public_html/sdev.apratifoods.asia/public/
5. Create new folder called "storage"
6. Go into public/storage/
7. Paste the copied folders

Result:
/public/storage/brands/logos/xxx.png (copied from /storage/app/public/brands/logos/xxx.png)
